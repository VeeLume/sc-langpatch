# Screenshots

Drop image files in this folder. The main `README.md` references these filenames — keep the names stable so the links don't break.

| File | What it should show |
|------|---------------------|
| `app-main.png` | The main app window after auto-detection: install list at the top, modules below, big "Patch" button. The "money shot" — first thing readers see. |
| `app-modules.png` | The Modules panel expanded — all toggles visible so people can see what's optional. |
| `app-results.png` | The Patch Results panel after a successful patch — counts per module, no errors. |
| `app-language-pack.png` | The Language Pack input area, with a community URL pasted in. Optional, only needed if you want to highlight the community-pack feature. |
| `ingame-component-grades.png` | In-game inspect view of a ship component (e.g. a cooler) showing the new `MIL1C Bracer`-style label. |
| `ingame-mission.png` | In-game mission terminal showing a mission description with the blueprint / reward / cooldown additions. |

## Capture tips

- Use Windows **Snipping Tool** (Win+Shift+S) for the app shots; it crops to the app window cleanly.
- For in-game shots, F12 (default Steam) or PrintScreen works. Crop to the relevant panel afterwards.
- PNG, not JPG — text stays crisp.
- 1200–1600px wide is plenty; the README renders them at content width.
- If you change a screenshot, keep the filename the same so README links don't rot.
